# GOMGOM 카드 생성 플레이북 (Cowork 예약 작업용)

이 문서는 Claude가 예약 작업 세션에서 **처음부터 끝까지 혼자** 수행하기 위한 절차서다.
역할: GitHub Actions 파이프라인이 만든 카피(JSON)를 힉스필드로 카드뉴스 이미지로 제작해 리포에 넣는다.

## 0. 트리거 확인
1. 브라우저(Claude in Chrome)로 열기: `https://raw.githubusercontent.com/moneybuilderjay-cmyk/gomgomi-pipeline/main/data/queue.json`
2. `items` 중 `status == "awaiting_render"`인 항목을 찾는다. 없으면 **작업 종료** (아무것도 하지 않음).
3. 각 대상의 카피 읽기: `https://raw.githubusercontent.com/moneybuilderjay-cmyk/gomgomi-pipeline/main/data/cardjobs/<item_id>.json`
   (동명 `.done`이나 `_urls.json`이 이미 있으면 이미 처리된 것 — 스킵)

## 1. 카드 구성 매핑 (content JSON → 카드 목록)
순서대로:
1. **표지 1장** — `cover_layout`에 따라 A(character)/B(bignum)/C(chat) 레이아웃 (아래 프롬프트 템플릿)
2. **본문 N장** — `cards[]` 순서대로, type별 레이아웃:
   - `info`/`invert` → 색변주 스카이카드 또는 다크 텍스트 카드 (invert면 스카이 필드)
   - `table` → 데이터 비교 (표는 스탯 그리드나 행 리스트로 재구성)
   - `bignum`/`fullbleed_num` → 풀블리드 대형 숫자
   - `checklist`/`steps`/`summary` → 체크리스트 카드
3. **인용 카드 1장** — `view_quote` 필드 사용 (없으면 카피에서 아저씨 관점 한 문장 도출)
4. **CTA 1장** — `cta.gift` 있으면 "댓글에 "곰곰" → {gift} 노션 DM", 없으면 저장+팔로우
총 장수 = 본문 + 3. 페이지 번호는 "01 / NN" 형식, 표지=01.

## 2. 힉스필드 생성 레시피
- 모델: `nano_banana_pro`, aspect_ratio `4:5`, 카드당 count 1
- **캐릭터 레퍼런스(필수)**: medias에 곰곰 아저씨 원본을 넣는다.
  - 1순위: `media_import_url`로 `https://raw.githubusercontent.com/moneybuilderjay-cmyk/gomgomi-pipeline/main/assets/character_v3/gomgom_hero.png` 임포트 → 반환된 media_id를 medias [{role:"image", value:<media_id>}]로
  - 프롬프트에 "THIS EXACT bear character (glasses, navy suit, sky-blue tie)" 명시
- **팔레트 고정**: 배경 #0A1526 / 카드 #12203A / 스카이 #5FA8F5 / 상승 초록 / 하락 핑크-레드 / 텍스트 화이트
- **모든 카드 공통 요소** (프롬프트에 매번 포함):
  - Top-left: "GOMGOM" + glowing sky-blue dot / Top-right: 페이지 번호
  - 배경: 다크 네이비 + 소재와 어울리는 사진 은은히 블렌드 (데이터센터/월가 야경/트레이딩 데스크/연준 건물 등 소재 맞춤)
  - 대형 워터마크 영단어 1개 (티커나 테마: META, DATA, VIEW, FLOW...) very low opacity
  - Footer: "@gomgomi.economy" / "기준 YYYY.MM.DD" (오늘 날짜)
  - 곰곰 아저씨 등장: 표지 A·CTA=대형, 그 외=코너 소형+말풍선 또는 원형 아바타+질문 말풍선(q)
- **한국어 문구는 반드시 따옴표로 정확히 지정**하고 프롬프트 끝에 "perfect Korean spelling exactly as given" 포함
- 레이아웃별 상세 프롬프트 예시는 `design/v3/힉스필드_풀세트_매니페스트.md`(로컬)와 아래 요약 참조:
  - 표지 A: pill 배지 + 2줄 대형 타이틀(핵심어 스카이) + 아저씨 대형 + 통념 질문 말풍선 + 하단 스카이 pill 티저(cover_teaser)
  - 표지 B: 상단 pill(cover_label) + 초대형 글로우 숫자(cover_num) + 서브(cover_sub) + 아저씨 코너 피크(cover_bubble)
  - 표지 C: 채팅 3버블(cover_chat: bear 아바타 ↔ "G" 아바타 스카이 버블) + 하단 타이틀 + "진실은 옆장에 →" pill
  - 인용: "GOMGOM's VIEW" pill + 대형 인용부호 + view_quote(핵심 구절 스카이) + "곰곰 아저씨 · 20년차 미국주식" 바이라인 + 아저씨 팔짱

## 3. 검수 (필수)
0. **생성 전 카피 에디팅**: cardjob의 문구를 그대로 쓰기 전에 한국어 자연스러움을 검토하고, 어색한 표현은 의미를 유지한 채 다듬어서 카드에 사용하라 (예: "왜 쌌는지"→"왜 싸졌는지", 이중적으로 읽히는 표현·번역투·기사 축약체 교정). 소리 내어 읽어 어색하면 고친다.
1. 생성된 각 카드를 브라우저로 열어(결과 rawUrl) **한국어 오탈자·문구 누락·레이아웃 확인**
2. 문제 있는 카드만 같은 프롬프트로 재생성 (프롬프트에 문제점 교정 지시 추가)
3. 캐릭터 얼굴/수트가 원본과 크게 다르면 재생성

## 4. 결과 커밋
1. `Write`로 로컬 파일 생성: `pipeline/data/cardjobs/<item_id>_urls.json`
   ```json
   {"item_id": "<item_id>", "urls": ["<카드1 rawUrl>", "...카드 순서대로"]}
   ```
2. GitHub 웹 업로드로 커밋: `https://github.com/moneybuilderjay-cmyk/gomgomi-pipeline/upload/main/data/cardjobs`
   (파일 선택은 file_upload 도구, 커밋 메시지 예: "cards: <item_id> urls")
3. Actions에서 **fetch-cards** 워크플로 실행: `https://github.com/moneybuilderjay-cmyk/gomgomi-pipeline/actions/workflows/fetch_cards.yml` → Run workflow
   → 성공 확인 (out/<topic_id>/card-N.jpg 커밋됨, 상태 rendered)
4. Actions에서 **pipeline** 워크플로 실행 → rendered 건이 텔레그램 승인 요청으로 발송됨
5. 로그에 "렌더 완료 → 승인 요청" 확인되면 작업 완료

## 5. 예외 처리
- 힉스필드 크레딧 부족/생성 실패 반복 → 시도 중단, 사용자에게 상황 보고
- queue.json에 awaiting_render가 있는데 cardjobs JSON이 없음 → 파이프라인 오류, 보고만
- GitHub 커밋 단계: 커밋 후 반드시 URL 경로로 커밋 성공 확인 (upload 페이지에 남아있으면 재시도 — 커밋 메시지 입력란 클릭이 한 번에 안 먹는 경우가 잦음)
